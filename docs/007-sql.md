# 007 - SQL Databases

## Overview

Spript provides a SQL manager supporting PostgreSQL, MySQL, MariaDB, H2, and SQLite via HikariCP connection pooling. All queries run on **virtual threads** and return Promises.

## Creating a Connection Pool

```typescript
const pool = sql.createPool({
    url: "jdbc:postgresql://localhost:5432/minecraft",
    user: "admin",
    password: "secret",
    poolSize: 10       // optional, default is HikariCP's (usually 10)
});
```

### Connection Strings by Database

```typescript
// PostgreSQL
const pgPool = sql.createPool({
    url: "jdbc:postgresql://localhost:5432/mydb",
    user: "postgres",
    password: "pass"
});

// MySQL
const mysqlPool = sql.createPool({
    url: "jdbc:mysql://localhost:3306/mydb",
    user: "root",
    password: "pass"
});

// MariaDB
const mariaPool = sql.createPool({
    url: "jdbc:mariadb://localhost:3306/mydb",
    user: "root",
    password: "pass"
});

// H2 (file-based)
const h2Pool = sql.createPool({
    url: "jdbc:h2:file:./plugins/Spript/database"
});

// SQLite (via H2 or sqlite driver)
const sqlitePool = sql.createPool({
    url: "jdbc:h2:file:./plugins/Spript/data;MODE=SQLite"
});
```

## Querying

Use `sql.query()` for SELECT statements:

```typescript
async function getPlayers() {
    const results = await sql.query(pool, "SELECT * FROM players WHERE active = ?", [true]);
    for (const row of results) {
        console.log(`${row.name} - ${row.score}`);
    }
}
```

The result is an array of objects where keys are column names:

```typescript
const rows = await sql.query(pool, "SELECT id, name, score FROM players ORDER BY score DESC LIMIT 10");

// rows is Array<{id: number, name: string, score: number}>
for (const row of rows) {
    msg(player, `<gold>${row.name}</gold> <gray>-</gray> <green>${row.score}</green>`);
}
```

## Executing Statements

Use `sql.execute()` for INSERT, UPDATE, DELETE:

```typescript
// Insert
const affected = await sql.execute(pool,
    "INSERT INTO players (uuid, name, score) VALUES (?, ?, ?)",
    [playerId, "Notch", 1000]
);
console.log(`Inserted ${affected} row(s)`);

// Update
await sql.execute(pool,
    "UPDATE players SET score = score + ? WHERE uuid = ?",
    [50, playerId]
);

// Delete
await sql.execute(pool,
    "DELETE FROM players WHERE last_login < ?",
    [someTimestamp]
);
```

## Parameterized Queries

Always use `?` placeholders — never concatenate user input:

```typescript
// SAFE — uses prepared statement
const result = await sql.query(pool,
    "SELECT * FROM users WHERE name = ?",
    [userInput]
);

// DANGER — SQL injection risk!
// const result = await sql.query(pool, "SELECT * FROM users WHERE name = '" + userInput + "'");
```

## Complete Example: Player Stats

```typescript
import { PlayerJoinEvent } from 'org.bukkit.event.player';
import { PlayerQuitEvent } from 'org.bukkit.event.player';
import { CommandSender } from 'org.bukkit.command';

const pool = sql.createPool({
    url: "jdbc:h2:file:./plugins/Spript/playerdata"
});

// Create table on startup
await sql.execute(pool,
    `CREATE TABLE IF NOT EXISTS player_stats (
        uuid VARCHAR(36) PRIMARY KEY,
        name VARCHAR(16),
        joins INT DEFAULT 0,
        last_login BIGINT
    )`
);

listen(PlayerJoinEvent, async (event) => {
    const player = event.getPlayer();
    const uuid = player.getUniqueId().toString();
    const name = player.getName();
    
    await sql.execute(pool,
        `MERGE INTO player_stats (uuid, name, joins, last_login)
         VALUES (?, ?, 1, ?)
         ON DUPLICATE KEY UPDATE
         name = ?, joins = joins + 1, last_login = ?`,
        [uuid, name, Date.now(), name, Date.now()]
    );
});

command("stats", async (sender, label, args) => {
    const playerName = args.length > 0 ? args[0] : sender.getName();
    const results = await sql.query(pool,
        "SELECT * FROM player_stats WHERE name = ?",
        [playerName]
    );
    
    if (results.length === 0) {
        msg(sender, `<red>Player "${playerName}" not found</red>`);
        return;
    }
    
    const stats = results[0];
    msg(sender, `<gold>=== Stats for ${playerName} ===</gold>`);
    msg(sender, `<green>Joins: ${stats.joins}</green>`);
    msg(sender, `<gray>Last login: ${new Date(stats.last_login).toLocaleString()}</gray>`);
});
```

## Lifecycle

- Connection pools persist across `/jsreload` — create them once
- Pools are closed on server shutdown
- You can create multiple pools for different databases

## Notes

- All SQL operations are **async** — use `await` or `.then()`
- Uses HikariCP for connection pooling (production-grade, battle-tested)
- Runs on **virtual threads** (Java 21+) — won't block the main thread
- Placeholders use `?` syntax (JDBC standard)
- Column names in results match your database schema
